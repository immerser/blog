hadoop集群安装

HDFS daemons are NameNode, SecondaryNameNode, and DataNode. YARN daemons are ResourceManager, NodeManager, and WebAppProxy. MapReduce Job History

通过脚本配置环境变量

  1. etc/hadoop/hadoop-env.sh
  2. etc/hadoop/mapred-env.sh
  3. etc/hadoop/yarn-env.sh

修改配置

  1. etc/hadoop/core-site.xml
  2. etc/hadoop/hdfs-site.xml
  3. etc/hadoop/yarn-site.xml
  4. etc/hadoop/mapred-site.xml

环境准备

操作系统使用centos7.2
hadoop版本使用hadoop-2.9.0 放置路径/apps/hadoop-2.9.0/

机器分配

一共需要四台机器进行集群环境的模拟

机器 namenode datanode journal node zookeeper zkfc resource manager data manager
node1 Y Y Y Y
node2 Y Y Y Y Y Y
node3 Y Y Y Y Y
node4 Y Y Y Y

192.168.59.101 node1\
192.168.59.102 node2\
192.168.59.103 node3\
192.168.59.104 node4

机器配置

  1. 安装java（省略）
  2. 安装zookeeper（省略）

  3. 配置机器用户
    在四台机器上分别配置用户和用户组 均为 Hadoop，并设计目录权限

  4. 使用hadoop用户登录 edits文件地址（配置在journal node上）路径为/data/hadoop/journal/

  5. 配置ssh免登录

安装必要软件

  1. java
  2. zookeeper
  3. hadoop

操作步骤

  1. 启动zokeeper集群

  2. 在其中一台初始化ZooKeeper集群中对应hadoop的元数据 (node1)

  3. namenode文件系统格式化(node1)

    只在其中一台上进行格式化,其他的namenode通过同步的方式进行
    必须在journalnode启动后进行

  4. 同步启动后的namenode(node1)上的数据到另一台namenode (node2) 在 node2上执行

  5. 依次启动所有的datanode (node4)

问题记录

  1. You may run zkfc on the node other than namenode.

    dfs.namenode.http-address.clusterxx.xx 配置地址信息不正确

  2. Automatic failover is not enabled for NameNode at node4/192.168.1.72:9820. Please ensure that automatic failover is enabled in the configuration before running the ZK failover controller.

    dfs.client.failover.proxy.provider.hadoop_cluster 未配置 注意是 集群名称不是namenode名称

  3. Unable to fetch namespace information from active NN
  4. 先启动 namenode 再启动 journalnode
  5. Can’t format the storage directory because the current directory is not empty

    停止journal node 然后删除 对应的edit目录和hadoop格式化目录重新格式化
    Problem binding to [node4:9820] java.net.BindException: Address already in use; For more details
    hdfs rpc地址和http地址相同

  6. Incomplete HDFS URI, no host: hdfs:

    hadoop里面配置的主机host名称不能带下划线 修改名称

  7. Incompatible clusterIDs in /tmp/hadoop-root/dfs/data: namenode clusterID = CID-9baaf04a-1a8d-46dc-b132-2fcf12bae7ed; datanode clusterID = CID-55ccfbe9-1751-4182-8be6-ed19a9940f90

    删除datanode工作目录重新启动
